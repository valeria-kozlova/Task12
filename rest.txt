REST — сокращение от английского Representational State Transfer — "передача состояния представления".

REST — это архитектурный стиль, основанный на веб-стандартах и протоколе HTTP.
По сути REST - набор из 6 правил (ограничений), которые учитываются при проектировании распределенной системы.
Договоренность взаимодействовать между частями web приложений с помощью простого протокола - это и есть REST.
Как оказалось позднее REST оказался удобным не только для связи фронтэнда с бекэндом, но и для общения между собой разных бекэндов,
что помогает делать модульные web приложения и даже микросервисные архитектуры. Мы можем получать данные от общедоступных сервисов в интернете,
а можем свои собственные веб-сервисы использовать в нескольких проектах одновременно.

В архитектуре, основанной на REST, все является ресурсом. Доступ к ресурсу осуществляется через общий интерфейс на основе стандартных методов HTTP.
В архитектуре на основе REST есть сервер REST, который обеспечивает доступ к ресурсам. Клиент REST может получать доступ к ресурсам REST и изменять их.
Каждый ресурс должен поддерживать стандартные операции HTTP. Ресурсы идентифицируются глобальными идентификаторами (обычно это URI).

Принципы REST:
1. Приведение архитектуры к модели клиент-сервер.
2. Отсутствие состояния.
3. Кэширование.
4. Единообразие интерфейса.
5. Слои.
6. Код по требованию (необязательное ограничение).

Методы PUT , GET , POST и DELETE обычно используются в архитектурах на основе REST.
GET определяет доступ для чтения к ресурсу без побочных эффектов (идемпотент).
PUT создает новый ресурс. Он также должен быть идемпотентным.
DELETE удаляет ресурсы. Операции идемпотентны. Их можно повторять, не приводя к другим результатам.
POST обновляет существующий ресурс или создает новый ресурс.

RESTful веб-сервисы.
Веб-службы RESTful основаны на методах HTTP и концепции REST. То есть сервисы, написанные с учетом ограничений REST.
Веб-служба RESTful определяет базовый URI для служб, поддерживаемые типы MIME (XML, текст, JSON, пользовательский).
Он также определяет набор поддерживаемых операций (POST, GET, PUT, DELETE).
